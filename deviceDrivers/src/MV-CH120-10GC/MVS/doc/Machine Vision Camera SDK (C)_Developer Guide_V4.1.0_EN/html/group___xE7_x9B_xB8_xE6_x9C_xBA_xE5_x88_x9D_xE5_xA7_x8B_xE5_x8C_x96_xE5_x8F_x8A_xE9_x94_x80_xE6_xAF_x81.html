◆ MV_CC_EnumerateTls()

MV_CAMCTRL_API int __stdcall MV_CC_EnumerateTls ( )

Get the supported transport layer.

Returns
Types and number of supported transport layers.
Remarks
The transport layers include GigE and U3V transport layers. GigE stands for industrial cameras that support GigEVision protocols, and U3V stands for industrial cameras that support USB3Vision protocols.

◆ MV_CC_EnumDevices()

MV_CAMCTRL_API int __stdcall MV_CC_EnumDevices ( IN unsigned int  nTLayerType,
IN OUT MV_CC_DEVICE_INFO_LIST *  pstDevList 
)

Enumerate all devices corresponding to specified transport protocol on the subnet.

Parameters
nTLayerType[IN] Transport layer protocol type. See parameter definition in CameraParams.h. For example, #define MV_GIGE_DEVICE 0x00000001 GigE device.
pstDevList[IN][OUT] Device list.
Returns
Return MV_OK on success, and return Error Code on failure.
Remarks
The memory of storing device list is allocated by the SDK and will be released and applied for during multiple-threaded API calling. It is recommended to avoid the multiple-threaded enumeration.
MV_GIGE_DEVICE sends out information of all GigE devices (including virtual GigE and GigE devices on GenTl), and MV_USB_DEVICE sends out information of USB devices (including virtual USB devices). 
 


Examples
ChunkData.cpp, Display.cpp, Events.cpp, ForceIP.cpp, Grab_ImageCallback.cpp, GrabImage.cpp, GrabImage_HighPerformance.cpp, GrabMultipleCamera.cpp, HighBandwithDecode.cpp, ImageProcess.cpp, MultiCast.cpp, ParametrizeCamera_FileAccess.cpp, ParametrizeCamera_LoadAndSave.cpp, ReconnectDemo.cpp, SetIO.cpp, SetParam.cpp, Trigger_Image.cpp, and Trigger_ImageCallback.cpp.

◆ MV_CC_IsDeviceAccessible()






      

        

          MV_CAMCTRL_API bool __stdcall MV_CC_IsDeviceAccessible 
          (
          IN MV_CC_DEVICE_INFO * 
          pstDevInfo, 
        

        

          
          
          IN unsigned int 
          nAccessMode 
        

        

          
          )
          
        

      





Check if the specified device can be accessed. 


Parameters

  

    
pstDevInfo[IN] Device information. 

    
nAccessMode[IN] Access type. 



Macro Definition  Value  Description   



MV_ACCESS_Exclusive  1  Exclusive permission, for other apps, the CCP register is only allowed to be read   



MV_ACCESS_ExclusiveWithSwitch  2  Preempt permission in mode 5, and then open with exclusive permission   



MV_ACCESS_Control  3  Control permission, for other apps, all registers are allowed to be read   



MV_ACCESS_ControlWithSwitch  4  Preempt permission in mode 5, and then open with control permission   



MV_ACCESS_ControlSwitchEnable  5  Open with control permission that can be preempted   



MV_ACCESS_ControlSwitchEnableWithKey  6  Preempt permission in mode 5, and then open with control permission that can be preempted   



MV_ACCESS_Monitor  7  Open device with reading mode, suitable under control permission   





  

  




Returns
Return true to indicate the device is accessible, and return false to indicate no permission or the device is offline. 


Remarks
You can read the device CCP register value to check the current access permission. 

 Return false if the device does not support the modes MV_ACCESS_ExclusiveWithSwitch, MV_ACCESS_ControlWithSwitch, MV_ACCESS_ControlSwitchEnableWithKey. Currently the device does not support the 3 preemption modes, neither do the devices from other mainstream manufacturers. 

 This API is not supported by CameraLink device.

◆ MV_CC_OpenDevice()






      

        

          MV_CAMCTRL_API int __stdcall MV_CC_OpenDevice 
          (
          IN void * 
          handle, 
        

        

          
          
          IN unsigned int 
          nAccessMode, 
        

        

          
          
          IN unsigned short 
          nSwitchoverKey 
        

        

          
          )
          
        

      





Open a device. 


Parameters

  

    
handle[IN] Device handle 

    
nAccessMode[IN] Access Right 



Macro Definition  Value  Description   



MV_ACCESS_Exclusive  1  Exclusive permission, for other apps, the CCP register is only allowed to be read   



MV_ACCESS_ExclusiveWithSwitch  2  Preempt permission in mode 5, and then open with exclusive permission   



MV_ACCESS_Control  3  Control permission, for other apps, all registers are allowed to be read   



MV_ACCESS_ControlWithSwitch  4  Preempt permission in mode 5, and then open with control permission   



MV_ACCESS_ControlSwitchEnable  5  Open with control permission that can be preempted   



MV_ACCESS_ControlSwitchEnableWithKey  6  Preempt permission in mode 5, and then open with control permission that can be preempted   



MV_ACCESS_Monitor  7  Open device with reading mode, suitable under control permission   





    
nSwitchoverKey[IN] Key for switching permissions. 

  

  




Returns
Return MV_OK on success, and return Error Code on failure. 


Remarks
You can find the specific device and connect according to inputted device parameters. 

 When calling the interface, the parameters nAccessMode and nSwitchoverKey are optional, and the device access mode is exclusive by default. Currently the device does not support the following preemption modes: MV_ACCESS_ExclusiveWithSwitch, MV_ACCESS_ControlWithSwitch, MV_ACCESS_ControlSwitchEnableWithKey.

 For USB3Vision device, the parameters nAccessMode and nSwitchoverKey are invalid. 


Examples
