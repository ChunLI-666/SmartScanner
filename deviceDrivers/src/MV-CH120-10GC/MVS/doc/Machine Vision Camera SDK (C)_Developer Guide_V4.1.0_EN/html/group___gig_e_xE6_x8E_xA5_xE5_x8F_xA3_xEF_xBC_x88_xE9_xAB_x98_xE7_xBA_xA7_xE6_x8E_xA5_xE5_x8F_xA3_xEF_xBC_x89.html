◆ MV_GIGE_SetDiscoveryMode()

MV_CAMCTRL_API int __stdcall MV_GIGE_SetDiscoveryMode ( IN unsigned int  nMode)

Set device ACK packet type.

Parameters
nMode[IN] Packet type: 0-unicast packet, 1-broadcast packet.
Returns
Return MV_OK on success, and return Error Code on failure.
Remarks
The API is supported only by GigE cameras.

◆ MV_GIGE_SetGvcpTimeout()

MV_CAMCTRL_API int __stdcall MV_GIGE_SetGvcpTimeout ( IN void *  handle,
IN unsigned int  nMillisec 
)

Set the GVCP command timeout.

Parameters
handle[IN] Device handle.
nMillisec[IN] Timeout period. Default value: 300, range: [0,10000], unit: ms.
Returns
Return MV_OK on success, and return Error Code on failure.
Remarks
After the device is connected, you can call this API to set the GVCP command timeout.

Get the GVCP command timeout.

Parameters
handle[IN] Device handle.
pnMillisec[IN][OUT] Timeout pointer, unit: ms.

Remarks
This interface is used to get the current GVCP timeout.

◆ MV_GIGE_SetRetryGvcpTimes()

MV_CAMCTRL_API int __stdcall MV_GIGE_SetRetryGvcpTimes ( IN void *  handle,
IN unsigned int  nRetryGvcpTimes 
)

Set the GVCP command retransmission times.

Parameters
handle[IN] Device handle.
nRetryGvcpTimes[IN] Retransmission times, ranging from 0 to 100.
Returns
Return MV_OK on success, and return Error Code on failure.
Remarks
When GVCP packet transmission is abnormal, you can call this API to set retransmission times to avoid the camera disconnection.
