Initialize Running Environment

  • Before running the frame grabber SDK, it is recommended to call MV_FG_Initialize() first to initialize the SDK running environment, so as to allocate the necessary resources in advance and save time for the follow-up calling operations.
  • To exit from the SDK, you can call MV_FG_Finalize() to release the resources occupied by the SDK. Note that after calling this API, all other APIs cannot be called except APIs for initializing and enumerating.
    The sample codes below show the process of initializing running environment:
    /*Initialize running environment*/
    MV_FG_Initialize();
    // TODO: Perform operations such as device control and image acquisition
    /*Clear the running environment and exit the program*/
    MV_FG_Finalize();
