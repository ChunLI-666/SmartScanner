◆ MV_FG_SetBufferNum()

MV_FGCTRL_API int __stdcall MV_FG_SetBufferNum ( IN STREAMHANDLE  hStream,
IN unsigned int  nBufferNum 
)

Set the number of internal buffers for the SDK.

Parameters
hStream[IN] Stream channel handle.
nBufferNum[IN] Number of buffers.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
The SDK has no internal buffer if this API is not called or the value of parameter nBufferNum is set to 0.
When there is no internal buffer, you need to request and register buffers to stream channels before starting image acquisition.
When the parameter nBufferNum is set to a value greater than 0, buffers will be registered internally by the SDK, and it is not allowed to register buffers to stream channels at this time.
The number of image buffers should be reasonably allocated. For Linux operating systems, if the number of image buffers is too large, then the system memory will be too low and the application will be killed by the operating system.
Examples
ChunkData.cpp, ConvertPixelType.cpp, Event.cpp, GrabImage.cpp, GrabImage_Callback.cpp, GrabMultipleCamera_Callback.cpp, and Reconnect.cpp.

◆ MV_FG_ReleaseFrameBuffer()

MV_FGCTRL_API int __stdcall MV_FG_ReleaseFrameBuffer ( IN STREAMHANDLE  hStream,
IN MV_FG_BUFFER_INFO *  pstBufferInfo 
)

Release the buffer information. This API is valid only when buffers are requested internally by the SDK. This API is used to release the image buffer which is no longer used.

Parameters
hStream[IN] Stream channel handle.
pstBufferInfo[IN] Buffer information.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
This API and MV_FG_GetFrameBuffer() should be used in pair. The image data (pstBufferInfo) obtained via MV_FG_GetFrameBuffer() should be released by this API.
Examples
ConvertPixelType.cpp, Event.cpp, GrabImage.cpp, and Reconnect.cpp.

◆ MV_FG_GetBufferChunkData()

MV_FGCTRL_API int __stdcall MV_FG_GetBufferChunkData ( IN STREAMHANDLE  hStream,
IN MV_FG_BUFFER_INFO *  pstBufferInfo,
IN unsigned int  nIndex,
OUT MV_FG_CHUNK_DATA_INFO *  pstChunkDataInfo 
)

Get the chunk data information of a buffer.

Parameters
hStream[IN] Stream channel handle.
pstBufferInfo[IN] Buffer information.
nIndex[IN] Chunk data index.
pstChunkDataInfo[OUT] Chunk data information.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
After getting the buffer information, this API must be called before calling the APIs MV_FG_ReleaseFrameBuffer() and MV_FG_QueueBuffer() to get valid information of the chunk data.
Examples
ChunkData.cpp.

◆ MV_FG_GetPayloadSize()

MV_FGCTRL_API int __stdcall MV_FG_GetPayloadSize ( IN STREAMHANDLE  hStream,
OUT unsigned int *  pnPayloadSize 
)

Get the image size of the stream channel.

Parameters
hStream[IN] Stream channel handle.
pnPayloadSize[OUT] Image size of the stream channel.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
The image size of the stream channel needs to be reacquired after the image-related parameters of the camera are changed, such as width, height, and pixel format.
Examples

◆ MV_FG_AnnounceBuffer()

MV_FGCTRL_API int __stdcall MV_FG_AnnounceBuffer ( IN STREAMHANDLE  hStream,
IN void *  pBuffer,
IN unsigned int  nSize,
IN void *  pPrivate,
OUT BUFFERHANDLE *  phBuffer 
)

Register buffer to the stream channel. This API must be called prior to acquiring images.

Parameters
hStream[IN] Stream channel handle.
pBuffer[IN] Image buffer address.
nSize[IN] Image buffer size.
pPrivate[IN] Private information address.
phBuffer[OUT] Buffer handle.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
The image buffer size is acquired by calling the API MV_FG_GetPayloadSize(). The private information is user-defined.
Examples
GrabImage_UserBuffer.cpp.

◆ MV_FG_RevokeBuffer()

MV_FGCTRL_API int __stdcall MV_FG_RevokeBuffer ( IN STREAMHANDLE  hStream,
IN BUFFERHANDLE  hBuffer,
OUT void **  pBuffer,
OUT void **  pPrivate 
)

Revoke buffer from the stream channel.

Parameters
hStream[IN] Stream channel handle.
hBuffer[IN] Buffer handle.
pBuffer[OUT] Image buffer address.
pPrivate[OUT] Private information address.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
Only buffers in unused queues can be revoked. You can allocate buffers to unused queues by calling the API MV_FG_FlushQueue().
Examples
GrabImage_UserBuffer.cpp.
