Detailed Description

APIs including getting XML files of frame grabbers and cameras, setting and getting device parameters, loading and saving parameters.

◆ MV_FG_GetXMLFile()

MV_FGCTRL_API int __stdcall MV_FG_GetXMLFile ( IN PORTHANDLE  hPort,
IN OUT unsigned char *  pData,
IN unsigned int  nDataSize,
OUT unsigned int *  pnDataLen 
)

Get XML file of frame grabbers/cameras.

Parameters
hPort[IN] Handle of the object to which the parameter belongs; the object can be either a frame grabber or a camera.
pData[IN][OUT] Address of the buffer in which the XML file is stored.
nDataSize[IN] Size of the buffer in which the XML file is stored.
pnDataLen[OUT] Length of the XML file.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.

◆ MV_FG_GetNodeAccessMode()

MV_FGCTRL_API int __stdcall MV_FG_GetNodeAccessMode ( IN PORTHANDLE  hPort,
IN const char *  strName,
OUT MV_FG_NODE_ACCESS_MODE *  penAccessMode 
)

Get the access mode of a node.

Parameters
hPort[IN] Handle of the object to which the parameter belongs; the object can be either a frame grabber or a camera.
strName[IN] Node name.
penAccessMode[OUT] Node access mode.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
To get the access mode of the EnumEntry type node, the input value for strName should be in the format "EnumEntry_NodeName_EnumEntryName".
For example, to get the access mode of the node MV_FG_PIXEL_TYPE_Mono8 of the enumeration MV_FG_PIXEL_TYPE, the strName should be "EnumEntry_PixelFormat_Mono8".

◆ MV_FG_ReadPort()

MV_FGCTRL_API int __stdcall MV_FG_ReadPort ( IN PORTHANDLE  hPort,
void *  pBuffer,
int64_t  nAddress,
int64_t  nLength 
)

Read device register.

Parameters
hPort[IN] Handle (frame grabber or camera).
pBuffer[IN] Used as a returned value to save the read-in memory value. (Memory value is stored in accordance with the big end and little end model.)
nAddress[IN] Memory address to be read, which can be obtained from the xml file of the device. It is a node value in the format of xxx_RegAddr.
nLength[IN] Length of the memory to be read
pnOutLength[OUT] Actual length of the returned memory.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
Access the device and read the data from a certain segment of register.

◆ MV_FG_RegisterEventCallBack()

MV_FGCTRL_API int __stdcall MV_FG_RegisterEventCallBack ( IN PORTHANDLE  hPort,
IN const char *  strEventName,
IN MV_FG_EventCallBack  cbEvent,
IN void *  pUser 
)

Register event callback function.

Parameters
hPort[IN] Handle of the object to which the parameter belongs; the object can be either a frame grabber or a camera.
strEventName[IN] Event name.
cbEvent[IN] Event callback function.
pUser[IN] User defined data.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
Time-consuming operations in the callback function will block the access to the subsequent event information.
This API is applicable to the callback of frame grabber, camera, and stream events. It registers callback functions by passing in different handles.

◆ MV_FG_RegisterExceptionCallBack()

MV_FGCTRL_API int __stdcall MV_FG_RegisterExceptionCallBack ( IN PORTHANDLE  hPort,
IN MV_FG_ExceptionCallBack  cbException,
IN void *  pUser 
)

Register exception information callback function.

Parameters
hPort[IN] Handle of the object to which the parameter belongs; the object can be either a frame grabber or a camera.
cbException[IN] Exception information callback function.
pUser[IN] User defined data.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
Time-consuming operations in the callback function will block the access to the subsequent exception information.
This API is applicable to the callback of frame grabber and camera exceptions. It registers callback functions by passing in different handles.
Currently, the disconnected event of the frame grabbers is not supported. The frame grabbers do not support hot swap, so disconnected events will not occur.
After the frame grabber is upgraded, you need to restart the PC to use it. A friendly reminder has been given.
Examples
Reconnect.cpp.
