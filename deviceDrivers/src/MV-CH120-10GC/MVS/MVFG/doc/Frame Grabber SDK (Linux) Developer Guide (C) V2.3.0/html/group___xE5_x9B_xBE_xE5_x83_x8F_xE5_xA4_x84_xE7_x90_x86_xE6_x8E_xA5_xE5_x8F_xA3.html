◆ MV_FG_HB_Decode()

MV_FGCTRL_API int __stdcall MV_FG_HB_Decode ( IN IMAGEHANDLE  hImage,
IN OUT MV_FG_HB_DECODE_PARAM *  pstDecodeParam 
)

Lossless decoding.

Parameters
hImage[IN] Image handle. You can use the handle of frame grabbers, cameras, or stream channels.
pstDecodeParam[IN][OUT] Lossless decoding parameter structure.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
Decode the lossless compressed stream data from the camera into raw data.
The API can also parse the watermark information of the real-time image of the current camera. (If the input lossless stream data is not from the current camera, or is not real-time stream, then exception might occur during watermark parsing.)
The the handle of the current camera is needed to parse watermark information. The frame grabber handle or the stream handle can only be used for lossless decoding.

◆ MV_FG_ReconstructImage()

MV_FGCTRL_API int __stdcall MV_FG_ReconstructImage ( IN IMAGEHANDLE  hImage,
IN OUT MV_FG_RECONSTRUCT_INFO *  pstReconstructInfo 
)

Reconstruct images.

Parameters
hImage[IN] Image handle. You can use the handle of frame grabbers, cameras, or stream channels.
pstReconstructInfo[IN][OUT] Reconstruction information.
Returns
Return MV_FG_SUCCESS on success, and return Error Code on failure.
Remarks
Image rotation, image flipping, and image splitting are supported, but with specific preconditions.
  • The pixel format supported by image rotation and image flipping are MV_FG_PIXEL_TYPE_Mono8, MV_FG_PIXEL_TYPE_RGB8_Packet, and MV_FG_PIXEL_TYPE_BGR8_Packed.
  • Image splitting supports all pixel formats, and should be used with the "MultiLightControl" node of the line scan camera. The node can set multiple exposure values. For example, set MultiLightControl=2, then the camera will overlap and merge the two images with two different exposure values into one image (its height is the sum of the two images) and send it to the upper application.
  • Call this API and pass in RECONSTRUCT_MODE_SPLIT_BY_LINE_2 to divide the image sent by the camera into two images, each of which corresponds to an exposure value. If an ordinary camera is used or the "MultiLightControl" node of the line scan camera is not enabled, then the image splitting is meaningless. The image will only be divided into 2, 3, and 4 images by line, and the height of each image becomes 1/2, 1/3, 1/4 of the original image (depending on enReconstructMode).
